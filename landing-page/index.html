Predicting Overuse Injuries in Runners

A machine learning approach using Apple Watch training load data and XGBoost to identify injury risk patterns.

System Implementation

Workouts are synced via HealthKit, processed in the mobile app, and sent to a FastAPI backend for analysis.

Data Collection

Extracts 12 weeks of historical workout data (distance, HR, frequency) via Apple HealthKit.

→

Mobile Client

React Native application handles user authentication and initial data formatting for the API.

→

Feature Engineering

Python backend calculates 12 predictive features (ACWR, Monotony, Z-Scores) from the 12-week history.

→

XGBoost Inference

A trained Gradient Boosting model predicts the probability of injury based on the engineered features.

Development Key Features

The project focuses on transforming raw GPS and Heart Rate data into commonly used training metrics from sports science literature.

HealthKit Integration

Developed using react-native-healthkit to pull 12 weeks of training. I implemented filters to ensure only running data is analyzed to maintain model accuracy.

Workload Calculations

Implemented the Acute:Chronic Workload Ratio (ACWR). The system detects training "spikes" by comparing the current 7 days against a 28-day baseline.

LLM Feedback Loop

Integrated a language model (Gemini 2.5 flash lite) to translate model outputs into advice. I used prompt engineering to ensure the AI considers the specific risk factors found in the training data.

Backend Pipeline

The FastAPI backend performs feature engineering. It uses rolling windows and expanding medians to create a personalized training baseline for each user.

Model Optimization

Used an XGBoost classifier with L1/L2 regularization to manage the small dataset size and prevent overfitting during the training phase.

State Management

Uses React Context and AsyncStorage. The system is designed to be "local-first," storing prediction history on the device rather than a remote database.
